# 그때살껄 (GeudaeSalkkel) — 서비스 기획안

> "그때 샀다면 지금쯤..."  
> 미국 ETF 적립식 백테스트 계산기

---

## 1. 서비스 개요

### 한 줄 정의
내가 과거 특정 시점부터 미국 ETF 포트폴리오를 꾸준히 적립했다면, 지금 얼마가 됐을지 계산해주는 서비스.

### 배경 & 인사이트
- "그때 살껄..." 은 주식 투자자라면 누구나 공감하는 감정
- 국내에 ETF 적립식 백테스트 서비스가 토스 앱 외에 한국어 웹 서비스로 제대로 된 것이 없음
- 미국 ETF 장기 투자 관심이 5060 세대까지 확산되는 추세
- 감성적 네이밍으로 바이럴 가능성 높음

### 슬로건 후보
- "그때 샀다면 지금쯤..."
- "후회는 계산 이후에"
- "내가 버텼다면 얼마였을까"

---

## 2. 타겟 유저

### 메인 타겟
- **4050 직장인 / 은퇴 준비층**
- 미국 ETF에 관심은 있지만 어디서 시작할지 모르는 사람
- QQQ, SCHD, SPY 같은 ETF 이름은 들어봤지만 수익률 계산을 못 해보는 사람
- "토스 앱에서 이런 기능 봤는데 웹에서도 되면 좋겠다" 는 사람

### 서브 타겟
- 2030 투자 입문자
- 이미 ETF 투자 중인 사람 (내 포트폴리오 검증용)
- 재무 유튜버 / 블로거 (콘텐츠 소스로 활용)

### 유저 페르소나
> 김철수 (56세, 직장인)  
> "요즘 QQQ가 좋다는 말을 많이 듣는데, 10년 전부터 매달 100만원씩 샀으면 지금 얼마였을지 궁금하다. 어디서 계산해봐야 하는지 모르겠다."

## 3. 핵심 기능 정의

### MVP (1단계 출시 필수 기능)

#### 3-1. ETF 포트폴리오 구성
- ETF 티커 검색 및 추가 (자동완성)
- 종목별 비중(%) 설정
- 비중 합계 100% 유효성 검사
- 지원 ETF: 주요 미국 ETF 100종 이상 (QQQ, SPY, SCHD, JEPI, VOO, VTI 등)

#### 3-2. 투자 조건 설정
- 적립 주기: 매일 / 매주 / 매월
- 1회 적립금액 (달러 기준)
- 투자 기간: 시작일 ~ 종료일 (단축버튼: 1년 / 3년 / 5년 / 10년)

#### 3-3. 백테스트 계산 엔진
- **실제 주가 데이터 기반** (Yahoo Finance API 연동)
- DCA(Dollar Cost Averaging) 방식으로 매 주기마다 매수 시뮬레이션
- 배당금 재투자 여부 선택 옵션

#### 3-4. 결과 리포트
- 핵심 지표 카드
  - 총 투자원금
  - 최종 평가금액
  - 총 수익률 (%)
  - 연평균 수익률 CAGR (%)
  - S&P 500 대비 초과 수익률
  - 총 배당금 수령액
  - 투자 횟수
- 수익률 그래프: 내 포트폴리오 vs S&P 500 (SPY) 비교 라인 차트
- 종목별 개별 수익률 테이블

#### 3-5. 결과 공유
- 결과 이미지 저장 (PNG 다운로드)
- 카카오톡 / X(트위터) 공유 버튼
- URL 공유 (쿼리스트링으로 포트폴리오 정보 인코딩)

### 2단계 기능 (회원 기능)
- 회원가입 / 소셜 로그인 (카카오, 구글)
- 내 포트폴리오 저장 & 불러오기
- 즐겨찾는 포트폴리오 관리

### 3단계 기능 (프리미엄)
- 개별 주식 포함 (ETF + 종목 혼합 포트폴리오)
- MDD (최대 낙폭) 분석
- 샤프 지수, 변동성 분석
- 세금 / 수수료 반영 시뮬레이션
- 환율 반영 원화 환산
- 월별 / 연도별 상세 리포트
- 인기 포트폴리오 랭킹 (커뮤니티)

## 4. 화면 구성 (페이지별)

### 4-1. 랜딩 페이지 `/`
- 히어로 섹션: 슬로건 + 간단한 계산기 미리보기 (인터랙티브)
- 사용 방법 3단계 설명 (ETF 선택 → 기간 설정 → 결과 확인)
- 인기 포트폴리오 예시 카드 (QQQ 10년, SCHD 5년 등 클릭 → 바로 계산)
- "그때 안 샀다면 얼마 손해봤나" 자극 카피 섹션
- CTA: "지금 계산해보기" 버튼

### 4-2. 계산기 페이지 `/calculator`
- 좌측 (입력 패널)
  - ETF 티커 검색 & 추가 (자동완성)
  - 종목별 비중(%) 설정
  - 적립 주기 / 금액 / 기간 설정
- 우측 (결과 패널) — 모바일은 하단 스크롤
  - 핵심 지표 카드 (원금 / 평가금액 / 수익률 / CAGR / 초과수익률)
  - **그래프 ① 누적 평가금액 비교 라인 차트** (메인)
  - **그래프 ② 월별 수익률 히트맵**
  - **그래프 ③ 종목별 기여도 도넛 차트**
  - 종목별 수익률 테이블
  - 공유 버튼 (이미지 저장 / 카카오 / X / URL 복사)

### 4-2-1. 그래프 상세 스펙

#### 그래프 ① 누적 평가금액 비교 (메인 라인 차트)
- X축: 날짜, Y축: 평가금액 (달러)
- 3개 라인: 내 포트폴리오(빨강) / S&P 500(파랑) / 현금 보유(회색)
- 시간축 하단에 주요 사건 마커 표시
  - 예: 코로나 폭락(2020.03), 금리인상(2022.03), AI 버블(2023.01)
- 특정 시점 호버 시 툴팁: 각 라인 평가금액 동시 표시
- "이때 팔았다면?" 인터랙션: 특정 시점 클릭 → "그 시점에 팔았다면 XX달러 손해/이익"

#### 그래프 ② 월별 수익률 히트맵
- X축: 월(1~12), Y축: 연도
- 셀 색상: 수익(초록 계열) / 손실(빨강 계열) / 강도에 따라 진하기 차등
- 한눈에 "어느 해, 어느 달이 좋았는지" 파악 가능
- 공유하기 좋은 비주얼 → 바이럴 유도

#### 그래프 ③ 종목별 기여도 도넛 차트
- 최종 수익에 각 ETF가 얼마나 기여했는지 비율 표시
- 호버 시 해당 종목 수익금액 / 수익률 툴팁
- 비중과 기여도 괴리가 클 때 직관적으로 확인 가능

### 4-3. 결과 공유 페이지 `/result` (공유용)
- URL 쿼리스트링으로 포트폴리오 정보 인코딩 → 링크 공유
- OG 이미지 자동 생성 (카카오톡 링크 미리보기용)
  - 포트폴리오 구성 / 수익률 / 기간 핵심 정보 포함
- "나도 계산해보기" CTA → 계산기로 유입
- 공유 카드 미리보기 + 다운로드 버튼

### 4-4. ETF 정보 페이지 `/etf/[ticker]` (2단계)
- 개별 ETF 소개, 구성 종목, 과거 수익률
- SEO 용도 (검색 유입)
- 예: `/etf/QQQ`, `/etf/SCHD`

### 4-5. 인기 포트폴리오 페이지 `/popular` (2단계)
- 지난주 가장 많이 조회된 포트폴리오 TOP 10
- 커뮤니티 바이럴 유도 콘텐츠
- 클릭 시 해당 포트폴리오 계산기로 바로 이동

## 5. 기술 스택 & 아키텍처

### 프론트엔드
| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 14 (App Router) | 아토네임과 동일, SEO 유리 |
| 언어 | TypeScript | 타입 안전성 |
| 스타일 | Tailwind CSS | 빠른 개발 |
| 차트 | Chart.js | 가볍고 충분 |
| 배포 | Vercel | 무료, 간편 |

### 백엔드 / 데이터
| 항목 | 선택 | 이유 |
|------|------|------|
| 주가 DB | Supabase | 선탑재 데이터 저장, 무료 티어, 추후 회원 기능도 동일 DB 사용 |
| 데이터 수집 (오픈 전) | Python 스크립트 1회 실행 | Yahoo Finance에서 50종목 × 10년치 일괄 수집 후 Supabase INSERT |
| 데이터 업데이트 (오픈 후) | n8n 스케줄러 | 매일 장 마감 후 당일 종가만 자동 추가 |
| DCA 계산 엔진 | Next.js API Route | 서버사이드 계산, CORS 문제 없음, Vercel 무료 |
| 인증 (2단계~) | Supabase Auth | 소셜 로그인 내장 |

### 아키텍처 흐름

```
[오픈 전 1회 — 데이터 선탑재]
Python 스크립트
    → Yahoo Finance에서 주요 ETF 50종목 × 10년치 일별 종가 수집
    → Supabase etf_prices 테이블에 INSERT
    → 완료 (이후 Yahoo Finance 실시간 의존성 없음)

[오픈 후 — 매일 자동 업데이트]
n8n 스케줄러 (매일 오전 7시 KST, 미국 장 마감 후)
    → Yahoo Finance에서 전일 종가 수집 (50종목)
    → Supabase etf_prices에 UPSERT

[유저 백테스트 요청]
유저 브라우저
    ↓ 티커 + 비중 + 기간 + 적립주기 전달
Next.js API Route (/api/backtest)
    ↓ Supabase에서 해당 종목 기간 주가 조회
DCA 계산 엔진 (서버사이드)
    ↓ 매 적립일마다 매수 시뮬레이션 → 누적 평가금액 계산
결과 JSON 반환
    ↓
브라우저 — Chart.js로 렌더링
```

### Supabase DB 테이블 구조

```sql
-- 주가 데이터 (선탑재)
etf_prices (
  ticker    TEXT,
  date      DATE,
  close     NUMERIC,
  PRIMARY KEY (ticker, date)
)

-- 지원 ETF 목록
etf_list (
  ticker    TEXT PRIMARY KEY,
  name_ko   TEXT,
  name_en   TEXT,
  category  TEXT
)
```

### n8n 역할 (매일 업데이트 스케줄러)

## 7. 바이럴 전략 (UI/UX)

### 핵심 원칙
"공유하고 싶은 결과물"을 만드는 것이 바이럴의 전부. 아토네임의 인스타 공유 카드처럼, 그때살껄도 **공유 카드**가 핵심 성장 엔진.

### 7-1. 공유 카드 (핵심 바이럴 장치)
결과 페이지에서 1클릭으로 생성되는 이미지 카드.

```
┌─────────────────────────────┐
│  😱 그때살껄...              │
│                              │
│  QQQ 50% + SCHD 50%         │
│  2015년부터 매달 $1,000 적립  │
│                              │
│  원금        $120,000        │
│  현재 평가액  $487,230 🚀    │
│  수익률       +306%          │
│  연평균       +15.2%         │
│                              │
│  S&P500이었다면 $298,000     │
│  👉 $189,230 더 벌었어요     │
│                              │
│        geudaesalkkel.com     │
└─────────────────────────────┘
```

- PNG 1클릭 다운로드
- 카카오톡 / X(트위터) / 네이버 밴드 공유 버튼
- OG 이미지 자동 생성 → 링크 공유 시 미리보기에 핵심 수익률 노출

### 7-2. 감정 자극 카피 전략
숫자보다 감정이 공유를 만든다.

| 상황 | 카피 예시 |
|------|-----------|
| 결과 상단 | "5년 전 시작했다면 지금 $XX 더 있었어요" |
| S&P500 초과 시 | "같은 기간 S&P500보다 $XX 더 벌었어요 🎉" |
| S&P500 미달 시 | "S&P500이 이겼네요. 다른 조합을 찾아볼까요?" |
| 현금 대비 | "그냥 통장에 뒀다면 $XX에서 끝났을 거예요" |
| 공유 유도 | "이 결과, 주식 모임에 공유해보세요 👀" |

### 7-3. 바이럴 유입 채널 전략

| 채널 | 전략 |
|------|------|
| 네이버 카페 | 미국주식/ETF 카페에 결과 공유 문화 형성. 직접 포스팅 초기에 시드 |
| 유튜브 | "QQQ 10년 적립했다면?" 쇼츠 제작 후 그때살껄 링크 삽입 |
| X(트위터) | 충격적인 수익률 카드 공유 → 금융 트위터 커뮤니티 타겟 |
| 카카오톡 | 투자 오픈채팅방 타겟. OG 이미지가 바로 보여야 함 |
| 인플루언서 | 재테크 유튜버에게 무료 프로 계정 제공 → 콘텐츠 소스로 활용 |

### 7-4. 인기 포트폴리오 랭킹
- 메인 페이지에 "이번 주 가장 많이 조회된 포트폴리오 TOP 5" 노출
- 클릭 한 번으로 해당 포트폴리오 바로 계산
- 트렌드 따라가는 심리 자극 (FOMO 효과)
- 예: "요즘 핫한 포트 — QQQ+SCHD 이 조합 많이 보더라"

### 7-5. "후회 유발" 인터랙션
- 계산기 입력 중 실시간으로 "현재까지 X년 X개월 중 매달 $1,000 넣었다면 $XX" 미리보기
- 결과 페이지에서 기간 슬라이더 조작 시 그래프 실시간 업데이트
- "1년만 더 일찍 시작했다면 +$XX 차이" 자동 계산 표시

## 12. 유저 리텐션 전략

### 핵심 문제
백테스트는 "한 번 보면 끝"이 되기 쉬움. 재방문을 만드는 장치가 없으면 DAU가 쌓이지 않아 광고/구독 수익 모두 한계에 부딪힘.

### 12-1. 포트폴리오 구독 알림 (2단계~)
- 저장한 포트폴리오 월간 성과 이메일 자동 발송
  - 예: "내 QQQ+SCHD 포트폴리오, 이번 달 +3.2% 업데이트됐어요"
- 구현: n8n 스케줄러 + Supabase 저장 포트폴리오 조합

### 12-2. 월간 시장 리포트 페이지 `/report/[YYYY-MM]`
- 매월 자동 생성: "이번 달 미국 ETF 수익률 TOP 10"
- SEO 효과 + 재방문 유도 동시에
- SNS / 이메일 뉴스레터로 배포

### 12-3. 리밸런싱 알림 (프로 전용)
- 저장 포트폴리오의 실제 비중이 목표 비중에서 5% 이상 벗어나면 알림
- 예: "QQQ 비중이 목표 50%에서 57%로 늘었어요. 리밸런싱 타이밍일 수 있어요"

### 12-4. 재방문 훅 콘텐츠
| 콘텐츠 | 주기 | 채널 |
|--------|------|------|
| ETF 월간 수익률 랭킹 | 매월 1일 | 웹 + 이메일 |
| 인기 포트폴리오 순위 변동 | 매주 월요일 | 웹 |
| "이번 주 가장 많이 조회된 포트폴리오" | 매주 | 웹 메인 |
| 시장 급등/급락 시 푸시 알림 | 이벤트성 | 이메일 |

## 13. 온보딩 플로우

### 13-1. 첫 방문 유저 여정
```
랜딩 페이지 도착
    ↓
히어로 섹션에서 "지금 바로 체험해보기" CTA
    ↓
[선택 A] 추천 포트폴리오 템플릿으로 시작
    → 클릭 한 번으로 결과 페이지 이동
    → "이게 내 포트라면?" 감성 자극

[선택 B] 투자 성향 퀴즈 (3문항)
    → Q1. 투자 목적? (성장 / 배당 / 안정)
    → Q2. 투자 기간? (3년 / 5년 / 10년+)
    → Q3. 위험 감수 수준? (안정 / 중립 / 공격)
    → 맞춤 포트폴리오 추천 + 바로 백테스트
```

### 13-2. 추천 포트폴리오 템플릿 (랜딩 노출)
| 템플릿 이름 | 구성 | 타겟 |
|------------|------|------|
| 안정 성장형 | SPY 60% + BND 40% | 리스크 싫어하는 4050 |
| 배당 인컴형 | SCHD 50% + JEPI 50% | 월배당 원하는 은퇴 준비층 |
| 나스닥 집중형 | QQQ 70% + QLD 30% | 공격적 성장 원하는 3040 |
| 황금 비율형 | QQQ 50% + SCHD 50% | 성장+배당 균형 |
| 올웨더형 | SPY 40% + GLD 20% + TLT 40% | 시장 중립 |

### 13-3. 빈 계산기 방지
- 계산기 페이지 첫 로드 시 기본값 세팅
  - 기본 포트폴리오: QQQ 50% + SCHD 50%
  - 기본 기간: 최근 5년
  - 기본 적립금: $500/월
- "이 설정으로 바로 계산해보기" 버튼 노출 → 진입 장벽 최소화

## 15. 데이터 신뢰도 장치

### 15-1. 데이터 출처 명시
- 모든 결과 페이지 하단: "주가 데이터 출처: Yahoo Finance / 배당 데이터 포함"
- 데이터 마지막 업데이트 시각 표시: "2025.04.09 07:00 KST 기준"
- 주말/공휴일 처리 방식 설명: "장 미개장일은 직전 영업일 종가 적용"

### 15-2. 계산 방식 투명화
결과 페이지 내 "계산 방식 보기" 토글:
```
- 매수 가격: 적립일 당일 종가 기준
- 배당금: 배당락일 종가에 배당금 재투자 (선택 시)
- 환율: 계산에 환율 미반영 (달러 기준)
- 수수료: 미반영 (실제 수익률과 차이 발생 가능)
- 데이터 오차: ±0.5% 이내 (분할/합병 이벤트 반영)
```

### 15-3. 면책 고지 (법적 필수)
- 모든 페이지 하단 고정: "본 서비스는 투자 참고용으로, 실제 투자 결과와 다를 수 있습니다. 투자 손실에 대한 책임은 투자자 본인에게 있습니다."
- 결과 페이지 상단: "과거 수익률이 미래를 보장하지 않습니다"
- 프리미엄 추천 기능: "AI 분석 참고용이며 투자 자문이 아닙니다" 명시

### 15-4. 신뢰 지표 노출
- 메인 페이지: "누적 백테스트 계산 횟수 XX,XXX회" (사회적 증거)
- 데이터 커버리지: "50종목 / 최대 15년치 데이터 보유"
- 업데이트 주기: "매일 미국 장 마감 후 자동 업데이트"

- 입력 필드 간격: 충분한 여백으로 오터치 방지
- 숫자 입력 시 숫자 키패드 자동 전환 (`inputmode="numeric"`)

### 16-2. 모바일 전용 UX 플로우
```
[모바일 계산기 플로우]
Step 1 화면 — 포트폴리오 구성
    (ETF 추가 + 비중 설정)
    ↓ "다음" 버튼

    (금액 + 기간 + 주기)

    (지표 카드 → 그래프 → 테이블)
```

- 카카오톡 공유 시 OG 이미지 미리보기 검증 필수
  - OG 이미지 권장 크기: 1200×630px
  - 핵심 수익률 숫자를 이미지 중앙 상단에 배치
- 네이티브 공유 API 활용 (`navigator.share`) — 모바일에서 앱 선택 팝업 자동 노출
- 이미지 저장: 모바일에서 롱프레스로 저장 가능하게 구현

### 16-4. 성능 최적화 (모바일 데이터 환경 고려)
